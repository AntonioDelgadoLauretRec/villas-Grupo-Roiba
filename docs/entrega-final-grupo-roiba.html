01

Resumen Ejecutivo

Se presenta el sitio web corporativo de Grupo Roiba (gruporoiba.com), una plataforma digital de alto nivel diseñada para posicionar a la firma como referente en dirección técnica y construcción de villas en Punta Cana y el Caribe.

El proyecto incluye:

  • Sitio web completo con 10+ páginas públicas y sistema bilingüe (ES/EN)
  • Blog editorial con 6 artículos y sección de artículos relacionados
  • Panel de administración completo con gestión de contenido, villas, blog, equipo y configuración
  • Sistema de captación de leads integrado con Supabase
  • Analítica web con Google Analytics / GTM
  • Diseño responsivo validado en desktop, Android e iOS
  • Optimización SEO con datos estructurados, sitemap y metadatos completos

03

Funcionalidades Implementadas

Sistema bilingüe (ES/EN)

Toda la web soporta dos idiomas con cambio instantáneo desde cualquier página. El sistema utiliza un LanguageContext personalizado con React Context API. El idioma se persiste en localStorage y se aplica sin recarga de página.

Diseño responsivo

Diseño adaptativo con breakpoints en 640px (sm), 768px (md), 1024px (lg) y 1280px (xl). Se han aplicado correcciones específicas para:

  • iOS Safari: viewport-fit=cover, -webkit-fill-available, safe-area-insets, momentum scroll, sticky positioning
  • Android: overflow prevention, tap highlight removal, text-size-adjust
  • Scroll containers: scroll-snap para cards de servicios y tabs de proceso en móvil

Animaciones y microinteracciones

  • Scroll reveal system con Intersection Observer
  • Ken Burns effect en el hero de la homepage
  • Marquee horizontal de logos/confianza
  • Botones con efecto slide dorado al hacer hover
  • Cards con elevación y escala de imagen al hover
  • Flip cards en la sección de valores (Nosotros)
  • Carrusel de proyectos con auto-scroll

WhatsApp flotante

Botón flotante de WhatsApp disponible en todas las páginas para contacto directo.

Cookie Consent

Banner de consentimiento de cookies conforme a normativa, con aceptar/rechazar.

Seguridad

  • Content Security Policy (CSP) configurado via middleware
  • Protección de rutas admin con autenticación Supabase
  • Row Level Security (RLS) en todas las tablas de la base de datos
  • Validación de datos con Zod en todas las acciones del servidor

Tipografía editorial

  • H1 (hero): Cormorant Garamond, clamp(28px–52px), blanco sobre imagen
  • H2 (secciones): Cormorant Garamond, clamp(1.6rem–2rem), con línea dorada decorativa
  • H3 (subsecciones): Cormorant Garamond medium, clamp(1.2rem–1.45rem)
  • Cuerpo: DM Sans light, 1.06rem, color #3D4A5C, interlineado 1.85
  • Listas: Viñetas doradas personalizadas
  • Pull quote: Serif italic con barra dorada lateral

Artículos relacionados

Cada artículo incluye una sección de "Continúe explorando" que muestra 3 artículos relacionados seleccionados por afinidad temática:

  1. Artículos de la misma categoría tienen prioridad
  2. Se completa con artículos de otras categorías si no hay suficientes
  3. El artículo actual se excluye siempre de la selección

La sección se ubica entre el cuerpo del artículo y el CTA final. Las cards son idénticas al estilo del listado del blog para mantener coherencia visual.

Sistema híbrido de contenido

Los artículos pueden provenir de dos fuentes:

  • Supabase (base de datos): artículos gestionados desde el panel de administración
  • Datos estáticos: 6 artículos hardcoded como fallback si Supabase no está disponible

El sistema intenta primero la base de datos. Si no encuentra el artículo o la conexión falla, recurre a los datos estáticos. Esto garantiza que el blog funcione siempre, independientemente del estado de la base de datos.

06

Gestión de Leads con Supabase

Captación

El formulario de contacto (/contacto) captura datos del interesado y los envía al endpoint /api/leads, que los almacena en la base de datos Supabase.

Base de datos

Supabase (PostgreSQL) gestiona todas las tablas del sistema:

  • admin_profiles: perfiles de administradores con roles
  • site_settings: configuración global (key-value JSONB)
  • villas / villa_images: catálogo de villas con galería
  • projects / project_images: proyectos con galería
  • blog_posts: artículos del blog
  • services / sub_services: servicios y sub-servicios
  • process_steps: fases del proceso de trabajo
  • testimonials: testimonios de clientes
  • team_members: miembros del equipo
  • company_values: valores corporativos
  • pois / attractions: puntos de interés y atracciones

Seguridad de datos

Todas las tablas están protegidas con Row Level Security (RLS):

  • Lectura pública solo para contenido publicado/visible
  • Escritura restringida a usuarios autenticados con perfil de admin
  • Función auxiliar is_admin() para verificar permisos

Migraciones

Tres archivos de migración SQL en supabase/migrations/:

  1. 001_admin_schema.sql: Admin, villas, proyectos, settings, RLS
  2. 002_seed_admin.sql: Instrucciones para crear usuarios admin
  3. 003_content_schema.sql: Todas las tablas de contenido
